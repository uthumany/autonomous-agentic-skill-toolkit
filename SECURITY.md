# Security Policy

## Reporting a Vulnerability

We take the security of our project seriously. If you discover a security vulnerability, please report it to us as soon as possible.

**Please DO NOT open a public GitHub issue.**

Instead, please send an email to [INSERT SECURITY CONTACT EMAIL HERE] with a detailed description of the vulnerability, steps to reproduce it, and any potential impact. We will acknowledge your report within 48 hours and provide a more detailed response within 5 business days.

## Disclosure Policy

We kindly request that you do not disclose any vulnerabilities publicly until we have had a chance to address them. We will work with you to coordinate a public disclosure once the issue has been resolved.

## Supported Versions

We only provide security updates for the latest stable release of the toolkit.

## Best Practices

- Keep your dependencies up to date.
- Follow secure coding practices.
- Regularly review your code for potential vulnerabilities.
